# AI 쇼핑 에이전트 개발 프롬프트 히스토리 (Taste Finder App)

다음은 본 "쇼핑 프라이빗 큐레이션 앱(Taste Finder)"을 처음 기획하고 개발, 개선하기까지 입력되었던 핵심 프롬프트(명령 및 요청)들의 흐름을 정리한 내용입니다. 이 프롬프트들을 참고하면 동일하거나 유사한 형태의 AI 에이전트 결과물을 단계적으로 구현할 수 있습니다.

---

### [Phase 1: 초기 컨셉 기획 및 웹앱 기본 구조 생성]
- 사용자가 직관적으로 두 개의 이미지 중 하나를 선택하면 취향을 분석하여 맞춤 상품을 추천해주는 '프라이빗 큐레이터' 형태의 웹앱을 만들고 싶어.
- 앱 컨셉: **"퍼스널 쇼퍼 기능"**처럼 사용자의 취향을 세련되게 파악할 것.
- 기본 디자인: **모노톤(블랙/화이트/그레이)**, 극도로 미니멀하고 **애플(Apple)이나 하이엔드 럭셔리 브랜드의 느낌**이 나도록 디자인해줘.
- 필수 포함 내용: 
  - 첫 화면에서 크게 "당신의 고유한 취향을 발견합니다."라는 카피를 노출하고 '스니커즈', '공간 가전', '테이블웨어' 등 카테고리 선택 버튼 제공.
  - 퀴즈 방식: "직관적으로 끌리는 쪽을 선택하세요"라는 문구와 함께 이미지 2개 중 택1 방식.
  - 추천 결과: 분석 중이라는 진행 이펙트를 짧게(약 2초) 노출한 뒤, 가장 잘 어울릴법한 제품 5가지를 리스트 형태로 띄워줘 (상품명, 간단 설명, 예상 매칭율, 사진 포함).
- 언어 지원: 일본어/한국어 동시 지원 (버튼으로 쉽게 전환되도록)
- 소스 구조: 모든 코드는 깔끔하게 모듈화 (index.html, styles.css, app.js 분리)

### [Phase 2: 이미지 수집 및 상품 데이터 연결 (자동화 스크립트)]
- 퀴즈 A/B 선택 이미지들과 추천 상품 이미지들이 실제 이미지로 보여야 해. 
- 파이썬 스크립트를 하나 작성해 줘서, 각 질문 컨셉(아방가르드, 미니멀 등)에 맞는 멋진 인터넷 화보/제품 이미지들을 실제 다운로드 받아 프로젝트 안의 `images` 폴더에 저장하게 해줘.
- 최종 추천 결과에 뜨는 5개의 상품(총 15개 유형 * 5개 = 75개 정도)에는 사용자가 이 상품을 실제로 일본 '야후 쇼핑(Yahoo Shopping Japan)'에서 검색해볼 수 있도록 연동하는 **`SHOP NOW` (구매하기) 버튼 링크**를 각각 생성해줘. (상품 제목을 url-encode하여 검색 결과 페이지로 이어지도록 설정할 것).

### [Phase 3: 디자인 디테일 및 사용성 개선 (UI/UX 퀄리티업)]
- (아이콘 추가) feather icons 같은 라이브러리를 써서, 선택 버튼들 옆에 작고 깔끔한 아이콘을 붙여줘. 
- 스크롤 디자인: 결과창이나 이미지 목록을 내릴 때, 애플처럼 부드럽고 여백(padding)이 넉넉해서 숨 막히지 않고 시원하게 스크롤할 수 있도록 해줘.
- 모바일 사이즈에서도 전혀 깨지는 부분 없이 깔끔하게 100% 최적화되게, 반응형(미디어 쿼리) 스타일을 추가해줘.

### [Phase 4: GitHub 리포지토리(Pages) 연동 및 배포 설정]
- 지금까지 내 로컬 컴퓨터에서 만든 이 파일들을, 인터넷에서도 누군가 접속할 수 있도록 **GitHub Pages**를 이용해서 배포하고 싶어.
- 로컬 `MY Work/shopping-ai-agent` 폴더와 `https://github.com/thinkfunker/shopping-ai-agent` 원격 리포지토리를 연결시키고 올려주는 터미널 명령어와 절차를 처음부터 끝까지 알려주고 자동으로 실행해줘.

### [Phase 5: 데이터 보완 및 예외 처리 (Fallback Logic)]
- 쇼핑 링크(일본 야후)로 넘어갈 때, 언어를 "한국어"로 바꾼 상태에서 퀴즈를 완료해도 해당 링크들은 여전히 일본어 검색 결과를 띄울 수 있도록 (일본 EC 연결 유지) 내부 스크립트 연결 주소 데이터를 동기화(Sync) 해줘.
- 검색어로 야후 쇼핑을 켰을때 간혹 "판매 종료이거나 없는 상품(404 에러나 0개 검색)"이 나와 당황스러워. `SHOP NOW` 버튼 아래에 작게 회색 글씨로 **`상품이 없나요? 비슷한 제품 찾기` (Find Alternatives)** 라는 보조 링크를 추가해서, 세부 모델명이 아닌 포괄적인 타이틀 키워드로 다시 안전하게 검색할 수 있도록 로직을 만들어 줘.

### [Phase 6: 카테고리 대규모 확장 (GOLF, ACCESSORIES, CASUAL)]
- 퀴즈 메인 화면에 **브랜드로 찾기(FASHION BRAND)** 대분류 버튼을 추가해 줘. 그리고 그 아래에 **골프(GOLF), 잡화(ACCESSORIES), 캐주얼(CASUAL)** 3가지 세부 카테고리 퀴즈를 새롭게 구축해줘.
- 각 카테고리당 10개씩의 퀴즈 문항 사진(A/B)을 구성할건데, 이번엔 "제품 이미지"가 아니고 각 유명 브랜드가 가진 고유의 감성이 담긴 **"광고 화보 캠페인 룩북/이미지"**를 다운로드 받아 퀴즈 화면에 띄워줘.
- 이 세 가지 브랜드별 분류는 저/중/고가 등 가격대를 의식하지마. **오로지 "사용자가 선호하는 룩, 스타일링 무드, 디자인 철학" 에 따라 4가지씩 다른 결과(예: 하드코어 스트릿, 미니멀, 고프코어 등)**가 나오도록 완전히 멋지게 네이밍하고 상품 기획을 구성해줘.
- 새롭게 만들어진 약 120장 이상의 이미지와 거대한 JSON 로직 덩어리를 `app.js`에 주입해주고 깃허브 메인 서버에 Push 해줘.

### [Phase 7: 서버 배포 트러블슈팅 (GitHub Pages 설정)]
- GITHUB 배포 페이지에서 404가 나거나 Build 오류, 버튼 먹통 현상이 일어나고 있어. 
- (해결 1) GitHub Jekyll 빌드 특수성 때문에 한글이나 띄어쓰기가 들어간 이미지 파일명에서 에러가 나는 것 같으니, **파이썬 파일을 통해 모든 다운받은 이미지의 이름을 안전한 알파벳 영문 해시 혹은 일련번호로 `rename` 하고 코드와 매핑해줘.**
- (해결 2) 파이썬 `venv` 가상환경 등 불필요한 거대 폴더들을 깃허브에서 삭제해주고 깔끔하게 연동해 줘.
- (해결 3) `app.js`의 객체 괄호(`}`) 문법 에러 및 기존 3개 카테고리(스니커, 가전, 테이블웨어) 로직 데이터가 누락된 것을 원상 복구 및 merge 해줘.
